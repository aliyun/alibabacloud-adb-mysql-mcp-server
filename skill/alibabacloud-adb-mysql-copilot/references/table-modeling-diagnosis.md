## 二、数据处理逻辑
- **并行调度**：并行触发上述 7 项诊断，待全部返回后汇总。
- **问题数映射**：诊断概览与详情表中 count1～count7 依次对应上表诊断项 1～7 的 `TotalCount`（或该项结果条数）。
- **报告日期**：填写报告生成日期时间戳（如 2026-03-10 19:35:40）。
- **状态列**：问题数 = 0 时填 ✅，问题数 > 0 时填该行等级符号（🔴 / ⚠️ / 💡）。
- **诊断概览**：同一张表中展示各诊断维度的**问题数**、**状态**、**详情摘要**。详情摘要从该项诊断结果抽取关键信息（每项取前 1～5 条：库.表 + 主要指标）；无问题则填「无」。
- **诊断详情**：各诊断项结果**分段展示**，不合并为一张总表。按诊断项 1～7 顺序，每项单独成小节（三级标题），该小节内仅嵌入该项的完整 Markdown 数据表格及对应修复动作；无问题时该小节可写「无异常」或省略表格。
